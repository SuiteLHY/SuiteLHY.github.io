从源仓库的分支(branch)拉取代码，同时合并到分支仓库的分支(branch)

操作步骤

  1. 在分支仓库的指定分支下，创建新的拉取请求(Pull request)

    补充说明：

    • 指定分支可以在第二步中比较变化(Comparing changes)时选择；

    • 此处预先选好指定分支的操作，只是作为习惯，也是个建议；

    • 预先选好指定分支的操作，有2个好处：

      1. 方便确认指定分支代码的情况；

      2. 避免出现忘记切换到指定分支的失误操作。

    • ⤷ 可选操作步骤① — 第2步

  2. （自动跳转 🡆 在源仓库的默认分支下）比较 分支之间的变化(Comparing changes)

    • ⤷ （自动跳转 🡆 ）默认情况下，源项目默认分支对应分支 作为base repository，分支项目指定分支 作为head repository。所以，此时我们需要将base repository与head repository对调。 ⤵

  3. 比较并处理完 所有 来自源仓库的分支的commit之后，将其作为 [ 分支仓库的分支下的 commit ] 进行提交。
